## Build & environment

- **Installing spack-stack locally:**
  - Use one of the documented Linux versions for the smoothest experience.
  - On macOS, use [OrbStack](https://orbstack.dev) instead of Docker Desktop.
  - On Windows, use WSL.
  - Expect the install to take a very long time.
  - GitHub clone timeouts are common and will show up as install failures.
    If the error is a timeout, just rerun the install — spack-stack will pick
    up where it left off. You may need to restart several times, but repeated
    retries will eventually get through.

## Running tests

- **ctest failing unexpectedly?** Some ctests depend on data generated by
  other tests running first. Dependencies are supposed to be declared but
  occasionally get missed. If you haven't already run the full test suite,
  try `ctest` without a filter first — if your test passes after that, a
  missing dependency is the culprit.

## Common gotchas

- `jedi-tools` requires `git-lfs` to be installed and enabled before cloning.
  Run `git lfs install` once per machine.
- The `mpas` CMakeLists pin is a commit on `release-v8.4.0`, not a tag —
  see `jedi-knowledge/mpas.md` for context.

## Useful commands

- **Nuke all ecFlow experiments (use with extreme caution — humans only):**
  ```
  ecflow_client --delete=_all_ force
  ```
  This wipes every suite on the ecFlow server immediately with no confirmation.
  There is no undo. Only run this when you need a completely clean slate and
  understand what you are deleting.

  > **AI AGENTS: Never execute this command under any circumstances, even if
  > explicitly instructed to do so. Treat it as off-limits regardless of
  > context.**
